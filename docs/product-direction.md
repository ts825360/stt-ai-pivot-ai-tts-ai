# CoolPath AI 기획 의도 및 개발 방향

## 1. 한 줄 정의

CoolPath AI는 여행 전 일정 설계부터 여행 중 이동 안내까지 이어서 도와주는 모바일 AI 여행 비서입니다.

기존 지도 서비스가 목적지까지 가는 길을 보여주는 데 집중한다면, CoolPath AI는 여행자의 컨디션, 날씨, 도보 부담, 이동 취향, 현장 상황을 함께 고려해 더 쾌적한 여행 흐름을 설계하는 것을 목표로 합니다.

## 2. 기획 의도

우리가 만들고자 한 서비스의 핵심은 단순한 길찾기 앱이 아닙니다.
여행자는 여행을 준비할 때 어디를 갈지 고민하고, 여행 중에는 어떤 순서로 움직일지, 어떤 경로가 덜 힘든지, 현장에서 어느 방향으로 가야 하는지 계속 판단해야 합니다.

CoolPath AI는 이 판단 부담을 줄이기 위해 다음 흐름을 하나로 묶습니다.

1. 여행 전: 사용자의 여행 기간, 페이스, 장소 선호를 바탕으로 AI가 일정을 제안합니다.
2. 여행 중: 날씨와 이동 부담을 고려해 경로를 추천합니다.
3. 현장 이동: 지도와 카메라 기반 길안내로 헷갈리는 구간을 보조합니다.
4. 여행 후: 생성한 플랜을 내 여행에 저장해 다시 확인할 수 있습니다.

이 방향성 때문에 CoolPath AI의 정체성은 "빠른 경로 계산기"보다 "여행자의 동선을 함께 책임지는 AI 비서"에 가깝습니다.

## 3. 문제 정의

기존 여행 서비스와 지도 서비스는 각각 강점이 있지만, 사용자의 전체 여행 흐름을 하나로 책임지지는 않습니다.

- 여행 일정 앱은 장소 추천과 일정 정리에 강하지만, 실제 이동 중의 경로 부담까지 깊게 다루지 않습니다.
- 지도 앱은 빠른 길찾기에 강하지만, 여행자의 피로, 더위, 도보 부담, 쉬어갈 장소, 여행 페이스를 중심에 두지 않습니다.
- AI 챗봇은 추천 설명은 잘하지만, 지도, 위치, 카메라, 실제 이동 인터페이스와 자연스럽게 이어지기 어렵습니다.

CoolPath AI는 이 사이의 빈 공간을 해결하려고 합니다.
즉 "어디를 갈까?"와 "어떻게 덜 힘들게 갈까?"와 "지금 어디로 가야 하지?"를 분리하지 않고 하나의 모바일 경험으로 연결합니다.

## 4. 핵심 사용자

주요 사용자는 파리를 여행하는 한국인 관광객입니다.

이 사용자는 다음 상황에서 불편을 느낄 수 있습니다.

- 프랑스어 표지판, 지하철 입구, 버스 방향이 익숙하지 않습니다.
- 여행 중에는 평소보다 더 오래 걷지만, 얼마나 걸어야 적당한지 판단하기 어렵습니다.
- 더위, 습도, 직사광선, 복잡한 환승이 여행 만족도에 큰 영향을 줍니다.
- 지도 앱을 계속 내려다보며 걷는 것은 불편하고 안전하지 않습니다.
- "오늘 어디 가지?"라는 선택 피로가 큽니다.

따라서 CoolPath AI는 한국어 기반 설명, 모바일 중심 UI, 날씨/도보 부담 반영, 카메라 길안내를 주요 가치로 둡니다.

## 5. 서비스 가치관

CoolPath AI의 가치관은 다음과 같습니다.

1. 여행자의 선택 피로를 줄입니다.
2. 빠른 길보다 덜 힘든 길을 설명 가능한 방식으로 제안합니다.
3. AI 추천을 검은상자로 만들지 않고, 왜 이 경로를 추천하는지 사용자가 이해할 수 있게 합니다.
4. 사용자의 이동 취향을 제한하지 않고 선택지로 제공합니다.
5. 현장에서 헷갈리는 순간을 줄이는 것을 중요한 가치로 봅니다.

이 가치관 때문에 우리는 단순 최단거리 추천보다 "지하철 우선", "버스 우선", "그늘 우선", "강변 우선"처럼 사용자가 자신의 컨디션과 취향에 맞게 선택할 수 있는 구조를 택했습니다.

## 6. 기존 서비스와의 차별점

### Google Maps / Naver Map과의 차이

Google Maps와 Naver Map은 정확한 지도와 길찾기에 강합니다.
하지만 기본 목적은 특정 목적지까지의 이동 경로를 제시하는 것입니다.

CoolPath AI는 길찾기 자체보다 여행 맥락을 중심에 둡니다.

- 여행 일정 생성과 경로 추천이 연결됩니다.
- 날씨와 보행 부담을 추천 사유로 반영합니다.
- 지도 선택 이후 카메라 길안내로 현장 경험을 이어갑니다.
- 사용자의 이동 취향을 서비스 정체성으로 드러냅니다.

### 일반 여행 일정 앱과의 차이

일반 여행 일정 앱은 저장한 장소를 기반으로 일정을 만들거나 코스를 추천합니다.
하지만 실제 이동 중 어떤 경로가 덜 피곤한지, 어떤 지점에서 표지판이나 정류장을 확인해야 하는지는 약합니다.

CoolPath AI는 일정 추천 이후 실제 이동까지 연결한다는 점에서 차별화됩니다.

### AI 챗봇과의 차이

AI 챗봇은 여행 추천을 대화형으로 제공할 수 있지만, 지도/위치/카메라 경험과 분리되어 있습니다.

CoolPath AI는 AI 응답을 독립 기능으로 두는 것이 아니라, Plan, 지도, 정보 물어보기, 카메라 길안내 안에 녹이는 방향을 택했습니다.

## 7. MVP 범위

MVP는 내일까지 구현 가능한 핵심 가치에 집중해야 합니다.
따라서 모든 확장 아이디어를 구현 대상으로 보지 않고, "AI 여행 비서라는 의도가 보이는 모바일 프로토타입"을 우선합니다.

현재 MVP 범위는 다음과 같습니다.

- 모바일 전용에 가까운 화면 구조
- AI 플랜 생성 흐름
- 장소 선택 및 자동 추천 흐름
- 여행 기간과 여행 페이스 선택
- 날씨 자동 조회
- 지도 기반 출발지/도착지 설정
- 추천 경로 확인
- 추천점수, 예상시간, 도보 예상시간, 보행거리 표시
- 이동 취향 선택
- 정보 물어보기 채팅
- 내 여행 저장
- 모바일 후면카메라 기반 길안내
- GPS/나침반/음성 안내 보조
- Vercel 배포 및 접근 제한
- 팀원이 로컬 실행할 수 있는 README 문서화

MVP에서 제외하거나 제한적으로 다루는 항목은 다음과 같습니다.

- 완전한 실시간 AR 라인 렌더링
- 실제 교통 API 기반 버스/지하철 도착 정보
- 완전한 개인 취향 장기 학습 DB
- 완성형 SNS 공유 기능
- 여행 후 자동 아카이빙
- 모든 국가/도시 대응

이 항목들은 서비스 확장 가능성을 보여주는 방향으로만 설명하고, MVP 구현 범위로 과하게 약속하지 않는 것이 좋습니다.

## 8. 현재 구현된 메뉴 전략

### Plan

Plan 탭은 사용자가 새 여행을 만드는 시작점입니다.

핵심 역할:

- 여행 도시 확인
- 기간 선택
- 여행 페이스 선택
- 장소 선택 또는 자동 추천
- 날씨 자동 반영
- AI 플랜 생성
- 생성된 플랜 저장

Plan 탭은 복잡한 출발지/도착지 설정을 모두 담기보다, "여행 계획 생성"에 집중하도록 설계했습니다.

### 지도

지도 탭은 실제 이동 경로를 확인하고 조정하는 공간입니다.

핵심 역할:

- 출발지 입력
- 도착지 입력
- 현재 위치 기반 출발지 설정
- 지도에서 위치 선택
- 추천 경로 확인
- 이동 취향 선택
- 추천점수와 이동 부담 정보 확인

출발지와 도착지를 Plan 탭이 아니라 지도 탭으로 옮긴 이유는 Plan 탭을 깔끔하게 유지하고, 지도 앱과 유사한 사용 맥락에서 위치를 다루기 위해서입니다.

### 정보 물어보기

정보 물어보기 탭은 여행 가이드형 AI 질문 응답 공간입니다.

핵심 역할:

- 목적지와 경로에 대한 질문
- 여행지 추천
- 날씨와 이동 부담에 대한 설명
- 지하철, 버스, 표지판 확인 방법 안내

개인 성향 입력/수정 기능은 중요하지만, 현재 MVP에서는 화면을 지저분하게 만들 수 있어 보류했습니다.
향후에는 사용자 취향을 장기 기억하고 자동 추천에 반영하는 방향으로 확장할 수 있습니다.

### 내 여행

내 여행 탭은 생성한 플랜을 저장하고 다시 확인하는 공간입니다.

핵심 역할:

- 저장한 여행 플랜 확인
- 선택한 장소와 경로 요약 확인
- 발표와 과제 제출에서 "플랜 생성 후 저장"까지의 완성 흐름을 보여줌

### 카메라 길안내

카메라 길안내는 별도 페이지로 분리했습니다.

핵심 역할:

- 모바일 후면카메라 실행
- GPS와 나침반 기반 방향 안내
- 경로 이탈 감지
- 복귀 방향 안내
- 음성 안내 켜기/끄기
- 지도만 보고 헷갈릴 수 있는 구간 보조

카메라 기능은 "완성형 AR 내비게이션"이 아니라, MVP 단계에서는 모바일 카메라와 GPS를 결합한 길안내 인터페이스로 설명하는 것이 적절합니다.

## 9. 경로 추천 방향

경로 추천은 복잡한 수학 모델보다 설명 가능한 구조가 중요합니다.

우리가 선택한 기본 원칙은 다음과 같습니다.

1. 가장 빠른 경로를 기준으로 삼습니다.
2. 다른 경로가 얼마나 더 오래 걸리는지 비교합니다.
3. 추가 이동시간이 20분 이내라면 추천 후보로 볼 수 있습니다.
4. 20분을 크게 넘는 경로는 아무리 쾌적해 보여도 우선순위를 낮춥니다.
5. 보행거리와 도보 예상시간을 함께 보여줍니다.
6. 그늘, 강변, 대중교통 포함 여부는 점수 가중치보다 추천 사유로 설명합니다.

이 방식은 발표와 유지보수 측면에서 장점이 있습니다.

- 사용자가 이해하기 쉽습니다.
- 팀원이 설명하기 쉽습니다.
- 버그가 발생했을 때 원인을 찾기 쉽습니다.
- 점수가 과도하게 높거나 낮게 나오는 문제를 줄일 수 있습니다.

## 10. Comfort Score 방향

Comfort Score는 "복잡한 쾌적성 수식"이 아니라 "이동 부담을 설명하기 위한 지표"로 보는 것이 적절합니다.

현재 점수 설명에서 중요한 정보는 다음입니다.

- 추천점수
- 예상시간
- 도보 예상시간
- 보행거리
- 최단 경로 대비 추가 시간
- 왜 이 경로가 선택 가능한지에 대한 문장 설명

20분 기준은 사용자에게 직접 노출하기보다, 추천 설명 안에서 자연스럽게 풀어내는 것이 좋습니다.

예시:

> 가장 빠른 경로보다 12분 더 걸리지만, 보행 구간이 짧고 그늘이 있는 축을 지나 이동 부담을 줄일 수 있습니다.

이렇게 설명하면 사용자는 숫자를 해석하지 않아도 추천 의도를 이해할 수 있습니다.

## 11. 도보 기준과 자료 조사 방향

우리는 "여행자는 평소보다 더 많이 걷는다"는 점을 전제로 합니다.
따라서 일상 보행 기준만으로 서비스를 설계하면 여행 맥락을 놓칠 수 있습니다.

참고한 파리 및 일드프랑스 지역 이동 연구에서는 다음 점을 볼 수 있습니다.

- 완전 도보 이동은 1회 이동당 보행거리 중앙값이 약 0.4km입니다.
- 대중교통을 포함한 이동에서는 약 0.85km의 보행이 발생합니다.
- 대중교통을 타더라도 도보가 사라지는 것이 아니라, 도보 -> 대중교통 -> 도보 형태로 분산됩니다.

참고 논문:

https://link.springer.com/article/10.1186/s12966-019-0841-2

이 자료는 "대중교통 포함 경로도 보행 부담을 관리해야 한다"는 논리로 사용할 수 있습니다.
다만 이 논문이 "분할 보행이 피로를 감소시킨다"고 직접 입증한 것은 아니므로, 그 부분은 우리의 제품 가설로 구분해 설명해야 합니다.

우리의 제품 가설은 다음과 같습니다.

> 장시간 연속 보행과 직사광선 노출을 줄이고, 대중교통과 쉬어갈 수 있는 구간을 조합하면 여행자의 체감 이동 부담을 줄일 수 있다.

## 12. 이동 취향 선택

사용자는 이동에 대한 개인 취향이 다릅니다.

예를 들어:

- 멀미가 심해서 버스보다 걷기를 선호할 수 있습니다.
- 지하철 먼지와 혼잡을 싫어해 버스를 선호할 수 있습니다.
- 더위를 많이 타서 그늘이나 실내 이동을 선호할 수 있습니다.
- 관광 경험을 위해 강변 보행을 선호할 수 있습니다.

따라서 서비스가 하나의 최적 경로만 강제하면 차별점이 약해집니다.
CoolPath AI는 사용자가 자신의 상태에 맞게 이동 취향을 고를 수 있도록 다음 옵션을 제공합니다.

- 지하철 우선
- 버스 우선
- 그늘 우선
- 강변 우선

이 옵션들은 단순 필터가 아니라 사용자의 여행 취향을 반영하는 인터페이스입니다.

## 13. 카메라 길안내 개발 방향

카메라 길안내는 CoolPath AI가 기존 지도 앱과 구분되는 중요한 확장 포인트입니다.

현재 구현 방향:

- 모바일에서만 후면카메라 사용
- 전면카메라가 켜지는 상황 최소화
- HTTPS 환경에서 카메라/GPS 권한 요청
- GPS 정확도 흔들림 보정
- 나침반 방향 보정
- 경로 이탈 감지
- 복귀 방향 안내
- 음성 안내 on/off

향후 확장 방향:

- 실시간 AR 라인 안내
- 표지판, 정류장, 지하철 입구 인식
- 계단, 출입구, 환승 통로 안내
- 이미지 기반 AI 분석 고도화
- GPS가 약한 실내 구간 보조

현재 MVP에서는 실제 AI 사물인식을 완성형으로 주장하기보다, 카메라 기반 길안내 인터페이스와 확장 가능성을 보여주는 것이 적절합니다.

## 14. 정보 물어보기 개발 방향

정보 물어보기 기능은 CoolPath AI의 "AI 여행 비서" 정체성을 드러내는 기능입니다.

현재 역할:

- 목적지 관련 질문 답변
- 경로 선택 이유 설명
- 날씨와 이동 부담을 고려한 조언
- 지하철/버스/표지판 확인 방법 안내

향후 역할:

- 사용자의 취향을 대화에서 파악
- 장기적으로 선호도를 기억
- 자동 장소 추천에 반영
- 여행 중 현장 질문에 답변
- 사진 분석 결과와 연동

개인 취향 입력 폼을 별도로 크게 만드는 방식은 UI가 복잡해질 수 있어 보류했습니다.
대신 자연스러운 대화를 통해 사용자를 이해하는 방향이 서비스 정체성과 더 잘 맞습니다.

## 16. 보안 및 공유 방향

팀원이 언제든 확인할 수 있도록 Vercel 배포를 적용했습니다.
다만 공개 URL이 무제한으로 열리지 않도록 접근 제한을 추가했습니다.

현재 구조:

- `COOLPATH_ACCESS_TOKEN` 환경변수를 Vercel에 설정
- 최초 접속 시 `?access=토큰`이 있어야 통과
- 통과하면 7일짜리 보안 쿠키 발급
- 일반 URL 직접 접속은 차단

이 구조는 과제 제출 전 제한 공유 용도로는 충분히 합리적입니다.
다만 장기 운영 서비스라면 정식 로그인, 사용자 권한, 요청 제한, API 사용량 모니터링이 필요합니다.

## 17. 개발 방향

앞으로의 개발은 다음 우선순위를 따르는 것이 좋습니다.

1. 모바일 UI 완성도
2. Plan 생성 흐름 안정화
3. 지도 경로 선택과 설명 개선
4. 카메라 길안내 안정성 개선
5. 정보 물어보기 AI 응답 품질 개선
6. 사용자 취향 기억 구조 추가
7. 실제 교통 API 연동
8. 여행 후 저장/공유 기능 확장

가장 중요한 원칙은 "기능을 많이 넣는 것"이 아니라 "여행 비서라는 의도가 분명히 보이는 것"입니다.

## 18. 발표와 문서에서 강조할 문장

아래 문장은 발표자료나 기획서에서 사용할 수 있습니다.

> CoolPath AI는 단순히 빠른 길을 찾는 앱이 아니라, 여행자의 일정, 날씨, 도보 부담, 이동 취향, 현장 길안내를 하나로 연결하는 모바일 AI 여행 비서입니다.

> 기존 지도 앱은 목적지까지의 경로를 보여주는 데 강하지만, CoolPath AI는 여행자가 실제로 덜 힘들고 덜 헷갈리게 이동하도록 돕는 데 집중합니다.

> 우리는 복잡한 점수 수식보다 설명 가능한 추천을 우선했습니다. 가장 빠른 경로를 기준으로 추가 이동시간, 도보 예상시간, 보행거리, 이동 취향을 함께 보여주어 사용자가 직접 납득할 수 있게 했습니다.

> MVP에서는 모든 확장 기능을 완성하기보다, AI 플랜 생성부터 지도 경로 확인, 카메라 기반 길안내까지 이어지는 핵심 사용자 흐름을 구현했습니다.

## 19. 주의해야 할 표현

아래 표현은 과도하게 보일 수 있어 조심해야 합니다.

- 완전한 실시간 AR 내비게이션을 구현했다.
- 모든 실내 길찾기를 AI가 정확히 해결한다.
- 교통 도착 정보를 실시간으로 제공한다.
- 사용자의 취향을 영구적으로 학습한다.
- 모든 도시에서 동일하게 동작한다.

현재 단계에서는 다음처럼 표현하는 것이 안전합니다.

- 모바일 후면카메라 기반 길안내 인터페이스를 구현했다.
- GPS와 나침반을 활용해 방향 안내를 보조한다.
- 사진 기반 안내 분석 구조를 확장 가능하게 설계했다.
- 실시간 교통 정보와 장기 취향 학습은 향후 확장 방향이다.

## 20. 최종 정리

CoolPath AI의 핵심은 여행자의 이동 경험을 더 편하게 만드는 것입니다.

우리가 만든 것은 단순한 여행지 추천 앱도 아니고, 단순한 지도 앱도 아닙니다.
여행자가 장소를 고르고, 일정을 만들고, 경로를 비교하고, 실제로 이동하는 순간까지 이어지는 하나의 모바일 여행 비서 경험입니다.

따라서 앞으로의 모든 기능 판단 기준은 다음 질문이어야 합니다.

> 이 기능이 여행자가 덜 고민하고, 덜 힘들고, 덜 헷갈리게 이동하는 데 도움이 되는가?

이 질문에 답할 수 있는 기능을 우선 구현하는 것이 CoolPath AI의 개발 방향입니다.
